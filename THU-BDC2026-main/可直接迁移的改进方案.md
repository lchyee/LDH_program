## 方案一：截面分位数排名（ROI 最高）

### 问题

原始 PE/PB 等估值特征有两个致命缺陷：

1. **行业间不可比**：银行 PE≈5，科技 PE≈50，把这两只股票的 PE 放进模型，模型看到的是"科技远比银行贵"，但实际含义完全不同
2. **极端值拉偏分布**：亏损股 PE 可达 10000+，导致 StandardScaler 被极值主导，大部分股票的 PE 被压缩到接近 0

### 方案

对每个交易日，计算每只股票在当日全部股票中的估值百分位排名：

```python
def add_cross_sectional_rank(df, val_cols=None, suffix='_分位'):
    """对估值列按日期计算截面百分位排名。

    必须在 StandardScaler 之前调用！
    百分位天然归一化到 [0,1]，不需要额外缩放。

    Args:
        df: 包含 '日期' 列和估值列的 DataFrame
        val_cols: 要排名的估值列名列表，默认 PE/PB/PS/PCF
        suffix: 排名列后缀

    Returns:
        new_cols: 新增的列名列表（需加入 feature_columns）
    """
    if val_cols is None:
        val_cols = ['市盈率TTM', '市净率MRQ', '市销率TTM', '市现率TTM']

    new_cols = []
    for col in val_cols:
        rank_col = col + suffix
        if col in df.columns and df[col].notna().any():
            # pct=True 返回 [0,1] 区间的百分位
            df[rank_col] = df.groupby('日期')[col].rank(pct=True).fillna(0.5)
        else:
            # 列不存在或全空，填 0.5（中位数）
            df[rank_col] = 0.5
        new_cols.append(rank_col)

    return new_cols
```

### 调用位置

```
特征工程（per-stock）→ concat → 补全缺失列 → 【这里】→ StandardScaler → 训练
```

### 效果对比

| 指标 | 原始 PE | PE 分位数 |
|------|---------|----------|
| 银行股 PE=5 | 绝对值3.7 | 百分位0.12（比88%的股票便宜） |
| 科技股 PE=55 | 绝对值55 | 百分位0.78（比22%的股票贵） |
| 跨行业可比 | 否 | 是 |
| 抗极端值 | 否（PE=10000拉偏Scaler） | 是（百分位上限1.0） |
| 量纲统一 | 否（4个指标4种量纲） | 是（全是[0,1]） |

### 注意事项

- **必须在 groupby('日期') 下计算**，不能跨日排名（不同日期的股票池可能不同）
- **必须放在 StandardScaler 之前**，百分位已经是 [0,1]，不需要再缩放（但放进去一起 Scaler 也不会出错，只是多此一举）
- 如果数据中没有 PE/PB 列（数据源不支持），此方案不适用——先确认数据源能否提供
- 排名列是**个股级特征**（同日每只股票值不同），对排序任务有区分度

## 方案三：检测并移除无效特征

### 问题

以下两类特征对横截面排序模型有害，必须移除：

1. **全零/常数列**：数据源缺失导致 fillna(0)，整列方差为 0
2. **同值列**：同一交易日对所有股票取值相同（如 SHIBOR、大盘指数等）

### 检测代码

```python
def detect_bad_features(df, feature_columns, date_col='日期'):
    """检测全零列和同值列，返回应移除的列名列表。

    Args:
        df: 训练数据 DataFrame
        feature_columns: 特征列名列表
        date_col: 日期列名

    Returns:
        zero_cols: 全零/常数列（方差=0，永远无信息）
        same_value_cols: 同值列（同日所有股票相同，对排序无区分度）
    """
    # 1. 全零/常数列：整列方差为0
    zero_cols = []
    for col in feature_columns:
        if col not in df.columns:
            zero_cols.append(col)
            continue
        if df[col].std() < 1e-10:
            zero_cols.append(col)

    # 2. 同值列：在同一交易日内，所有股票取值相同
    same_value_cols = []
    for col in feature_columns:
        if col not in df.columns or col in zero_cols:
            continue
        # 按日期分组，看组内标准差是否全为0
        within_day_std = df.groupby(date_col)[col].std()
        if (within_day_std < 1e-10).all():
            same_value_cols.append(col)

    print(f"全零/常数列 ({len(zero_cols)}): {zero_cols}")
    print(f"同值列 ({len(same_value_cols)}): {same_value_cols}")
    print(f"  → 同值列在同日对所有股票相同，对排序零区分度")
    print(f"  → 建议全部移除，可减少 {len(zero_cols) + len(same_value_cols)} 维")

    return zero_cols, same_value_cols


def clean_feature_columns(feature_columns, zero_cols, same_value_cols):
    """从特征列表中移除无效列。"""
    bad = set(zero_cols + same_value_cols)
    cleaned = [c for c in feature_columns if c not in bad]
    print(f"特征维度: {len(feature_columns)} → {len(cleaned)} (移除 {len(bad)})")
    return cleaned
```

### 调用位置

```python
# 在特征工程之后、Scaler之前
zero_cols, same_cols = detect_bad_features(processed, feature_columns)
feature_columns = clean_feature_columns(feature_columns, zero_cols, same_cols)
```

### 常见同值特征黑名单

以下是A股数据中常见的"同值特征"，如果你的数据包含这些，直接从 feature_columns 中删除：

| 特征 | 来源 | 为什么同值 |
|------|------|----------|
| SHIBOR_ON/1W/1M | 银行间拆借利率 | 全市场统一利率 |
| QVIX | 50ETF隐含波动率 | 全市场统一指标 |
| 涨停家数/连板均值/封单资金 | 全市场统计 | 全市场统一统计量 |
| 上证指数/沪深300点位 | 大盘指数 | 全市场统一价格 |
| CPI_同比/PMI/M2_同比 | 宏观经济数据 | 月频全市场统一 |
| 北向资金净买入 | 港股通 | 2024年8月起已停止披露 |

**唯一例外**：如果你的模型不是做横截面排序，而是做**时序预测**（预测大盘明天涨跌），那这些宏观/市场特征是有用的。

## 方案四：softmax 权重分配替代等权 0.2

### 问题

赛制要求输出 Top5 股票及权重（权重和 ≤ 1）。等权 0.2 忽略了模型对不同股票的置信度差异——模型可能对第1名非常确信（分数 0.9），对第5名不太确定（分数 0.1），但两者权重相同。

### 方案

用 softmax 将预测分数转化为权重：

```python
import numpy as np

def predict_with_softmax_weights(model, sequences, stock_ids, top_k=5, temperature=1.0):
    """预测 TopK 股票并用 softmax 分配权重。

    Args:
        model: 训练好的模型
        sequences: 输入序列 [N, L, F]
        stock_ids: 股票代码列表
        top_k: 选取前几只
        temperature: softmax 温度，越大权重越均匀，越小越集中于第1名
                     推荐 0.5~2.0，1.0 为标准 softmax

    Returns:
        top_stocks: TopK 股票代码列表
        weights: 对应权重（和为1）
    """
    with torch.no_grad():
        scores = model(sequences).squeeze().cpu().numpy()  # [N]

    # 排序取 TopK
    order = np.argsort(scores)[::-1]
    top_indices = order[:top_k]
    top_stocks = [stock_ids[i] for i in top_indices]
    top_scores = scores[top_indices]

    # softmax 分配权重
    # temperature 越小，越集中于最高分；越大，越接近等权
    exp_scores = np.exp(top_scores / temperature)
    weights = exp_scores / exp_scores.sum()

    return top_stocks, weights
```

### 温度参数选择

| temperature | 效果 | 适用场景 |
|-------------|------|---------|
| 0.1 | 接近 one-hot，第1名拿0.95+ | 非常确信模型排序时 |
| 0.5 | 略集中，第1名约0.35 | **推荐**：适度集中 |
| 1.0 | 标准 softmax | 均衡 |
| 2.0 | 接近等权0.2 | 不确信排序时 |
| ∞ | 完全等权0.2 | 退化为原始方案 |

### 输出示例

```
temperature=0.5 时：
股票    分数    权重
A       0.90    0.38
B       0.75    0.27
C       0.60    0.19
D       0.45    0.11
E       0.30    0.05
```

## 方案五：predict-评估日期对齐检查

### 问题

如果 predict.py 的输入数据包含比训练截止日更晚的数据，模型会基于"未来"的市场环境选股，但评估的是训练截止日之后的 5 天——时间错位导致选出的股票在评估期表现差。

### 检查代码

```python
def verify_date_alignment(predict_data, test_data, train_end_date):
    """验证预测基准日与评估窗口的日期对齐。

    Args:
        predict_data: predict.py 使用的数据
        test_data: score_self.py 使用的测试数据
        train_end_date: 训练集截止日期（字符串或Timestamp）

    规则：
        predict_data 的最后一天 == 训练集最后一天 == test_data 第一天之前
    """
    predict_latest = pd.to_datetime(predict_data['日期']).max()
    test_start = pd.to_datetime(test_data['日期']).min()
    train_end = pd.to_datetime(train_end_date)

    print(f"预测基准日: {predict_latest.date()}")
    print(f"训练截止日: {train_end.date()}")
    print(f"评估窗口开始: {test_start.date()}")

    if predict_latest > train_end:
        gap = (predict_latest - train_end).days
        print(f"⚠️ 日期错位！预测基准日比训练截止日多 {gap} 天")
        print(f"   模型基于 {predict_latest.date()} 的环境选股")
        print(f"   但评估在 {test_start.date()} 进行")
        print(f"   → 应该用训练截止日当天的数据做预测")
        return False
    elif predict_latest < train_end:
        print(f"⚠️ 预测基准日早于训练截止日，可能丢失近期信息")
        return False
    else:
        print(f"✅ 日期对齐正确")
        return True
```

### 修复方法


### 方案 A：行业哑变量（简单模型适用）

```python
def add_industry_dummy(df):
    """将行业分类转为哑变量列，每列表示"是否属于该行业"。

    优点：不增加模型参数，任何模型都能用
    缺点：28个行业→28个新特征，且无法表达行业间相似性

    if '行业名称' not in df.columns:
        return df, []
```

### 行业内截面排名（更精细的分位）

如果有了行业信息，可以计算**行业内**分位而非全市场分位，更合理：

```python
def add_industry_rank(df, val_cols=None):
    """行业内截面排名：同行业内比较，比全市场排名更合理。"""
    if val_cols is None:
        val_cols = ['市盈率TTM', '市净率MRQ', '市销率TTM', '市现率TTM']

    new_cols = []
    for col in val_cols:
        rank_col = col + '_行业分位'
        if col in df.columns and '行业名称' in df.columns:
            df[rank_col] = df.groupby(['日期', '行业名称'])[col].rank(pct=True).fillna(0.5)
        else:
            df[rank_col] = 0.5
        new_cols.append(rank_col)
    return new_cols
```

## 方案六深度分析：行业概念到底有没有用？

### 先说结论

在本比赛设定（沪深300选Top5，5天评估窗口）下：

| 用法 | 推荐 | 理由 |
|------|------|------|
| 行业 Embedding（改模型结构） | ❌ | 加~18K参数，小数据过拟合 |
| 行业哑变量（加特征列） | ⚠️ 低优先级 | 28维换行业信息，性价比一般 |
| 行业内截面分位（改特征工程） | ✅ 推荐 | 零模型改动，和全市场分位互补 |
| 行业概念本身 | ⚠️ 别抱太大期望 | 5天窗口太短，行业效应来不及体现 |

### 为什么行业 Embedding 不推荐

我们的项目加入了行业 Embedding（`nn.Embedding(28, 16)` + `Linear(144, 128)`），增加了约 18,592 个参数。加入后：

| 指标 | 加入前（原始基线） | 加入后（当前代码） |
|------|-----------------|-----------------|
| 测试得分 | 0.043 | 0.007~0.0264（无法复现0.043） |
| 训练 final_score | ~0.17 | 0.23 |
| best epoch | 6 | 39 |

训练指标变好了，但测试变差了——这是典型的**参数多了过拟合**。18K 参数在 ~400 个交易日 × 300 只股票的数据量下太多了。模型学到了训练期的行业-收益关联，但这种关联在测试的5天里不存在。

### 为什么行业内分位值得试

全市场分位和行业内分位回答的是不同的问题：

| 问题 | 全市场分位回答 | 行业内分位回答 |
|------|-------------|-------------|
| 这只股票便宜吗？ | "比全市场88%的股票便宜" | "在银行里排中间" |
| 估值是否异常？ | 能发现全市场低估值洼地 | 能发现行业内被低估的个股 |
| 适合什么策略？ | 价值策略（找便宜的买） | 行业轮动+选龙头 |

**A股现实**：低估值策略在银行、地产等传统行业长期有效，但在科技、新能源等成长行业，PE高不代表贵。行业内分位更合理，因为它告诉你"在同行里算不算贵"。

### 具体数值对比

以2026年某交易日为例（虚构但合理）：

| 股票 | 行业 | PE | 全市场分位 | 行业内分位 |
|------|------|-----|----------|----------|
| 工商银行 | 银行 | 4.8 | 0.05（极便宜） | 0.45（银行里排中间） |
| 招商银行 | 银行 | 6.2 | 0.10（很便宜） | 0.70（银行里偏贵） |
| 宁德时代 | 电力设备 | 35.0 | 0.72（偏贵） | 0.40（电力设备里偏便宜） |
| 比亚迪 | 汽车 | 28.0 | 0.65（偏贵） | 0.55（汽车里中等） |
| 贵州茅台 | 食品饮料 | 22.0 | 0.55（中等） | 0.80（食品饮料里偏贵） |

关键观察：
- 工商银行全市场分位0.05看起来"最便宜"，但银行里它只排中间 → 全市场分位被行业结构扭曲了
- 招商银行全市场分位0.10也"很便宜"，但在银行里偏贵 → 银行业内它不是最优选择
- 宁德时代全市场分位0.72"偏贵"，但电力设备里偏便宜 → 行业内更有价值

### 实战建议：两种分位都保留

```python
def add_all_ranks(df, val_cols=None):
    """同时添加全市场分位和行业内分位，让模型自己选择。"""
    if val_cols is None:
        val_cols = ['市盈率TTM', '市净率MRQ', '市销率TTM', '市现率TTM']

    new_cols = []

    # 全市场截面分位
    for col in val_cols:
        rank_col = col + '_分位'
        if col in df.columns and df[col].notna().any():
            df[rank_col] = df.groupby('日期')[col].rank(pct=True).fillna(0.5)
        else:
            df[rank_col] = 0.5
        new_cols.append(rank_col)

    # 行业内截面分位
    if '行业名称' in df.columns:
        for col in val_cols:
            rank_col = col + '_行业分位'
            if col in df.columns:
                df[rank_col] = df.groupby(['日期', '行业名称'])[col].rank(pct=True).fillna(0.5)
            else:
                df[rank_col] = 0.5
            new_cols.append(rank_col)

    return new_cols
```

这样每个估值指标产生2列（全市场+行业内），4个指标共8列。全市场分位捕捉"绝对便宜/贵"，行业内分位捕捉"相对便宜/贵"。两者信息互补，相关性<1.0。

### 行业信息的局限（为什么别抱太大期望）

**1. 5天评估窗口太短**

A股行业轮动周期通常为2-8周。5天窗口内，行业效应可能还没来得及体现，或者被个别股票的异动覆盖。即使模型正确判断了"当前银行强"，选出的银行股在那5天也可能被大盘拖累。

**2. 每行业样本量太少**

300只股票 ÷ 28行业 ≈ 10只/行业/天。行业内排名只有10个样本，统计意义很弱。某些小行业（如综合金融）可能只有2-3只，分位几乎没有区分度。

**3. 行业分类是静态的**

申万行业分类不随时间变化，但公司的业务会变化（转型、并购）。一只股票的行业标签可能已不准确。

**4. 模型已隐式学到部分行业效应**

同行业股票的价格走势高度相关，量价特征（Alpha158中的CORR、BETA等）已经隐式捕捉了行业联动。显式加行业信息的增量可能有限。

### 如果队友模型一定要用行业 Embedding

如果你队友的模型较大（参数量>1M）、数据量充足（>3年），可以尝试行业 Embedding，但注意以下几点：

1. **Embedding 维度要小**：8维足够，别用16维。28个行业×8维=224个参数，远小于16维的448个

2. **拼接位置要正确**：放在时序编码之后、股票间交互之前
   ```
   时序编码 → 特征聚合 → [拼接行业Embedding] → 跨股注意力 → 排序头
   ```
   不要把行业 Embedding 放进时序编码器——行业是静态属性，不随时间变化

3. **必须设 padding_idx=0**：无行业信息的股票填0，Embedding(0)输出全零向量，不影响其他股票

4. **保存行业词表**：训练时构建的 industry_vocab（行业名→索引映射）必须保存下来（joblib.dump），预测时加载使用。否则训练和预测的行业索引不一致，等于给了错误信号

5. **验证集中不要重新构建vocab**：直接复用训练集的 vocab，否则行业索引不一致

## 方案七：时间衰减样本权重（扩展数据时使用）

### 适用场景

当你把训练数据扩展到更早的年份时（如从2年扩展到5年），旧数据的分布可能与当前市场差异较大。时间衰减让模型更关注近期数据，但旧数据仍提供微弱的泛化信号。

### 实现代码

```python
# 1. 在数据集构建时，为每个样本（每个日期）计算权重

import numpy as np

def compute_time_weights(sample_dates, time_decay=0.5, half_life_days=365):
    """计算时间衰减权重。

    Args:
        sample_dates: 每个样本对应的日期列表（pd.Timestamp）
        time_decay: 衰减系数，0.5表示1年半衰期
            - 0.5: 1年前权重0.5，2年前0.25，3年前0.125
            - 0.7: 衰减更慢，旧数据影响更大
            - 1.0: 不衰减，等权
        half_life_days: 半衰期对应的天数，默认365

    Returns:
        weights: numpy array, 形状 (num_samples,)
    """
    if time_decay >= 1.0 or len(sample_dates) == 0:
        return None

    max_date = max(sample_dates)
    weights = np.array([
        time_decay ** ((max_date - d).days / half_life_days)
        for d in sample_dates
    ], dtype=np.float32)

    print(f"时间衰减权重: decay={time_decay}, "
          f"范围 [{weights.min():.4f}, {weights.max():.4f}], "
          f"均值 {weights.mean():.4f}")
    return weights


# 2. 在训练循环中，将权重乘到每个样本的 loss 上

for i in range(batch_size):
    # ... 计算每个样本的 loss ...
    loss = criterion(pred, target)

    # 应用时间衰减权重
    if sample_weights is not None:
        loss = loss * sample_weights[i]

    batch_loss = batch_loss + loss
```

### 衰减系数选择

| time_decay | 1年前权重 | 2年前权重 | 3年前权重 | 适用场景 |
|-----------|----------|----------|----------|---------|
| 0.3 | 0.09 | 0.01 | 0.001 | 强衰减，只关注近期 |
| 0.5 | 0.25 | 0.06 | 0.02 | 中等衰减（推荐） |
| 0.7 | 0.49 | 0.24 | 0.12 | 弱衰减，旧数据仍有影响 |
| 1.0 | 1.0 | 1.0 | 1.0 | 不衰减 |

### ⚠️ 警告

我们实验中发现，扩展到 2021 年的数据即使加了 time_decay=0.5，测试得分仍然下降（0.0264→0.0164）。**如果你的数据只有2-3年，不建议扩展**。这个方案只适合"数据已经扩展了、且旧数据分布与当前差异不大"的场景。

## 避坑清单

### ❌ 不要做的事

1. **不要在 feature_columns 中声明数据不存在的列** → fillna(0) 后全零列污染 Scaler 和模型
2. **不要把同值特征放进排序模型** → SHIBOR、QVIX、大盘指数等，同日所有股票值相同，对排序零贡献
3. **不要用 akshare 的宏观接口** → CPI/PMI/M2 接口不稳定，反复失败
4. **不要盲目扩展旧数据** → 2021-2023 市场风格与当下差异太大，拉低模型
5. **不要用过强的正则化** → dropout>0.3 或 weight_decay>1e-3 对小模型伤害大于帮助
6. **不要让 predict 用 merged CSV 的最新日期** → 可能导致预测-评估日期错位

### ✅ 推荐做的事

1. **PE/PB/PS/PCF + 截面分位** → baostock 内置字段，1行代码，3.8倍提升
2. **训练前检查特征有效性** → `df[col].std() > 0` 和 `groupby('日期').std() > 0`
3. **predict 用 train.csv** → 日期与评估窗口天然对齐
4. **softmax 权重** → 让高置信度选股获得更大权重
5. **行业内分位排名** → 比全市场排名更合理（需行业数据）
